1//! # Bootstrap address supply — breaking Kademlia's chicken-and-egg
2//!
3//! A fresh node has no peers and Kademlia needs peers to find peers. The
4//! [`BootstrapProvider`] trait supplies the initial list of addresses to contact so
5//! that first exchange can happen; after that
6//! [`crate::client::peer_tracker`] takes over and the bootstrap list stops mattering.
7//!
8//! Two implementations ship with the crate:
9//! - [`crate::transport::bootstrap_provider::dnssec_bootstrap_provider`] — production
10//!   path, resolves DNSSEC-validated TXT/A records under the domains in
11//!   [`crate::tools::config::BOOTSTRAP_DOMAINS`].
12//! - [`crate::transport::bootstrap_provider::manual_bootstrap_provider`] — explicit
13//!   address list, used by tests and by private deployments that want to pin seed nodes.
14
15/// Supplies the seed peer addresses a node contacts first when it comes online.
16///
17/// Joining a hashiverse network is a chicken-and-egg problem: Kademlia needs peers to find
18/// peers. A `BootstrapProvider` hands the client a starter list — hard-coded domain names,
19/// a file on disk, a DNS TXT record, whatever the deployment prefers — and the client uses
20/// those to make its first RPC. Once any successful peer exchange happens the
21/// [`crate::client::peer_tracker::PeerTracker`] takes over and the bootstrap list stops
22/// mattering.
23///
24/// This trait is deliberately narrow so it is easy to stub in tests
25/// (`ManualBootstrapProvider`) or point at a custom source in a private deployment.
26#[async_trait::async_trait]
27pub trait BootstrapProvider: Send + Sync {
28    async fn get_bootstrap_addresses(&self) -> Vec<String>;
29}
