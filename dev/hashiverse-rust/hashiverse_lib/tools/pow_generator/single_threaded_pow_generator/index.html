Single-threaded PoW search that works on every target.

No rayon, no spawn_blocking, no Web Workers — just pow_generate_with_iteration_limit called serially. WASM clients use this directly (with a relaxed pow_min); native callers can use it in unit tests where setting up a thread pool would be overkill.

Despite the simplicity, this generator does real PoW work — it is not a no-op stub.
