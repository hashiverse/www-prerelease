§Kademlia-style local peer set

PeerTracker is the client’s working memory of the network: a deduplicated list of crate::protocol::peer::Peer records loaded from BUCKET_PEER, validated, and indexed by XOR distance so crate::client::peer_tracker::peer_iterator::PeerIterator can walk them in order of closeness to any target crate::tools::types::Id.

Responsibilities:

  • Bootstrap — when the tracker is empty (new install, wiped storage) it calls BootstrapV1 against seed domains from crate::tools::config::BOOTSTRAP_DOMAINS to obtain a starting set of peers.
  • Freshness — stale peers (missed announces, failed RPCs) are demoted or dropped; fresh ones from gossip (AnnounceV1) or from RPC responses are folded in.
  • Flush — peer mutations set peers_need_flush; the outer client batches those into periodic writes to storage so every RPC response doesn’t trigger a disk write.
