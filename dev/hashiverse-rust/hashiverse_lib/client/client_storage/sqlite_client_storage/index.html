§SQLite-backed ClientStorage implementation

File-backed persistence using rusqlite. One table per crate::client::client_storage::client_storage::BUCKETS entry, each with (key TEXT PRIMARY KEY, value BLOB, last_accessed INTEGER). Opened with WAL mode (concurrent readers while a writer commits) and PRAGMA synchronous = NORMAL — safe for local cache use, not for power-loss durability.

A parking_lot::Mutex serialises all SQLite access so we never have two goroutines hitting the same connection at once. This is the store the native server binary and the desktop reference client use.
